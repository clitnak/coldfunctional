What is it?

ColdFunctional is a core language extension for Railo, inspired by Underscore, Ruby, Scala, and Clojure. It adds more functional programming language support to the coldfusion language, by providing more built-in, higher-order functions. It is a buffet-style extension, meaning, you don't have to install every new function.

NOTE: Railo is a great CFML engine (best out there, IMO). Unfortunately there is an issue where Railo built-in functions override user-defined functions. That means that if you have any user-defined functions with the same name as a function provided by Coldfunctional, whether authored by you or used in a framework that your app depends on, this extension may break it. Fortunately, this extension allows you to rename functions on install, to help you avoid conflicts. For example, If you are using Wirebox, you may want to rename the map method to something else (like _map) during Coldfunctional installation, in order to avoid this conflict.

Collection Functions

...ya know, for arrays, structs, and even queries. Each of these collection functions belongs to a function family. All functions in a family has essentially the same functionality, but it may have a more expressive name or different semantics. It may operate on different data structures (array, structs, queries). Some functions in a function family are already built into Railo. This extension tries to fill in the gaps.

Each function family

each applies a function to each element in a collection.
Member Functions
  • array.each( function(value:any) { return void; } )
  • struct.each( function(value:any) { return void; } )
  • query.each( function(record:struct) { return void; } )
Explicit Functions
  • ArrayEach( array, function )
  • StructEach( struct, function )
  • QueryEach( query, function )
Implicit Function
  • each( array, function )
  • each( struct, function )
  • each( query, function )

Map function family

map applies a given function element-wise to a collection of elements and returns a collection of results. There are three ways to to use the map
Member Functions
  • array.map( function(value:any) { return newValue; } )
  • struct.map( function(value:any) { return newValue; } )
  • query.map( function(record:struct) { return newRecordStruct; } )
Explicit Functions
  • ArrayMap( array, function )
  • StructMap( struct, function )
  • QueryMap( query, function )
Implicit Function
  • map( array, function )
  • map( struct, function )
  • map( query, function )

Filter function family

filter selects all elements in the in a collection where element satisfies a predicate function.
Member Functions
  • array.filter( function(value:any) { return boolean; } )
  • struct.filter( function(key:any,value:any) { return boolean; } )
  • query.filter( function(record:struct) { return boolean; } )
Explicit Functions
  • ArrayFilter( array, function )
  • StructFilter( struct, function )
  • QueryFilter( query, function )
Implicit Function
  • filter( array, function )
  • filter( struct, function )
  • filter( query, function )

Reduce function family

reduce will reduce a collection to a single value, applying the provided reduce function.
Member Functions
  • array.reduce( function(value:any, value:any) { return boolean; } )
  • struct.reduce( function(value:any, value:any) { return boolean; } )
Explicit Functions
  • ArrayReduce( array, function )
  • StructReduce( struct, function )
Implicit Function
  • reduce( array, function )
  • reduce( struct, function )
NOTE: query.reduce was not implemented, because it did not seem very useful to reduce a query to a single row. If you do think this would be useful, let me know.

Function Functions

Functions that operate on other functions to produce new functions. Each function belongs in a family similar to the collection functions, though these families are smaller, consisting of a vanilla explicit function and a member function. NOTE: the member functions will not work until this feature is implemented in the Railo core. Until then, you can use still use the explicit functions.

Once function family

once returns a function that will only be called once. In other words, the result is cached.
Member Functions
  • function.once()
Explicit Functions
  • functionOnce( function )

Once is moderately useful for a component as well. Railo has built-in function caching. For most, if not all, use cases function caching may be the more logical choice over using once Here is how you do it anyways, in case you crave the more expressive nature of higher-order functions. 


<cfcomponent>
	<cffunction name="getIncome">
		<cfreturn getMonthlyIncome()*12 />
	</cffunction>
	<cfset getIncome = getIncome.once() />
</cfcomponent>

Partial function family

The partial function, when provided a function and some arguments, will return a function that has the arguments already applied. If the resulting function is called, it will only need the remaining arguments, if any. Partial application of arguments can be done via named or unnamed arguments, by passing an array or struct.
Member Functions
  • function.partial( [] )
  • function.partial( {} )
Explicit Functions
  • functionPartial( function, [] )
  • functionPartial( function, {} )

Here are a few examples of how to use partial 


function add(x,y) {
	return x + y;
}

function power(x,y) {
	return x^y;
}

addOne = functionPartial( add, [1] );
five = addOne( 4 );

square = functionPartial( power, {y: 2} );
twentyfive = square( 5 );
